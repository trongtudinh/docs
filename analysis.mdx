## Mục tiêu

Trang này cung cấp cái nhìn tổng quan về dự án tài liệu (Mintlify) trong thư mục `docs/`, bao gồm cấu trúc thư mục, luồng điều hướng, và cách bạn có thể chỉnh sửa hoặc mở rộng nội dung.

## Kiến trúc tổng thể

- Nền tảng: [Mintlify](https://mintlify.com/) – dùng MDX để viết nội dung và `docs.json` để cấu hình.
- Thư mục gốc tài liệu: `docs/`
- Các phần chính:
  - Trang chính: `index.mdx`
  - Bắt đầu nhanh & phát triển: `quickstart.mdx`, `development.mdx`
  - Essentials (các tính năng cốt lõi của Mintlify): trong thư mục `essentials/`
  - API Reference: `api-reference/` (tự động sinh từ `openapi.json` và các trang endpoint)
  - AI Tools: `ai-tools/` (hướng dẫn dùng các công cụ AI để hỗ trợ viết tài liệu)

## Điều hướng (navigation)

Điều hướng được định nghĩa trong `docs.json`:

- Tabs chính: 1 tab "Docs"
- Nhóm "Getting Started": liên kết tới `quickstart`, `development`, và trang này `analysis`.
- Nhóm "Essentials": liên kết tới các trang trong thư mục `essentials/`.
- Nhóm "API Reference":
  - `api-reference/introduction`
  - Nhóm con "Endpoints": `api-reference/endpoint/{get,create,delete,webhook}`
- Nhóm "AI Tools": `ai-tools/{cursor, claude-code, windsurf}`

Lưu ý: Các đường dẫn trong điều hướng không cần phần mở rộng `.mdx`.

## API Reference

- OpenAPI đặt tại: `api-reference/openapi.json`
- Các trang endpoint:
  - GET /plants → `api-reference/endpoint/get.mdx`
  - POST /plants → `api-reference/endpoint/create.mdx`
  - DELETE /plants/{id} → `api-reference/endpoint/delete.mdx`
  - Webhook minh họa → `api-reference/endpoint/webhook.mdx`

Mintlify sẽ dùng frontmatter `openapi: 'METHOD /path'` trong các trang endpoint để hiển thị chi tiết API từ file OpenAPI.

## Phát triển cục bộ

1. Cài CLI:
   ```bash
   npm i -g mint
   ```
2. Chạy preview trong thư mục `docs/` (nơi có `docs.json`):
   ```bash
   mint dev
   ```
   Truy cập: http://localhost:3000

## Quy tắc & mẹo

- Mỗi khi thêm trang mới (.mdx), cần cập nhật `docs.json` để trang hiển thị trong sidebar.
- Có thể lồng nhóm (nested groups) để tổ chức nhiều cấp.
- Không dùng thư mục tên `api` ở cấp cao nhất (xem `essentials/navigation.mdx`).
- Kiểm tra link hỏng:
  ```bash
  mint broken-links
  ```

## Cách mở rộng

- Thêm chuyên mục mới: tạo thư mục mới và thêm các `.mdx`, rồi cập nhật `docs.json`.
- Tuỳ chỉnh giao diện: cập nhật `colors`, `logo`, `favicon` trong `docs.json` và file assets tương ứng.
- Tích hợp mạng xã hội: cấu hình trường `footer.socials` trong `docs.json`.

## Thay đổi gần đây (liên quan trang này)

- Cập nhật `docs.json` để:
  - Sửa đường dẫn API endpoints trỏ tới `api-reference/endpoint/...`
  - Loại bỏ mục SDKs không tồn tại và thay bằng các nhóm có thật
  - Thêm trang "Phân tích dự án" (trang này) vào nhóm "Getting Started"

Nếu bạn cần mở rộng thêm, hãy tạo issue hoặc cập nhật trực tiếp theo cấu trúc trên.
